文件上传之后，创建相应的知识库数据集，启动后台任务


# 关于文件数据集相关的后台任务拆分
首先是对整个文件进行训练的任务，比如上传文件的时候，修改分片设置的时候，需要使用。
另外，如果在修改某个文件分片内容的时候，或者修改整个知识库的向量模型的时候，只需要对分片进行重新向量化就可以了。


# 文件训练的后台任务的工作流程
如果数据集是本地文件，那么首先是文档解析，解析出文档内容和相应的图片，
将解析出来的文本内容保存到buffer_rawtexts表中，
将相应的图片保存到图片表images表中。
然后任务再根据数据集的分片设置，将文档内容进行分片，保存到数据集分片表dataset_datas中。
再根据分片，拆分成索引文本，因为为了不将含有强语义相关的章节分片，保持章节不被拆分，但是向量模型又普遍不支持2048个token，所以需要对分片进行进一步的拆分，再进行向量索引化。当然这个是做的非常细的情况下才需要，如果要求没有那么高，完全可以不加进一步拆分。
数据集表需要一个标记，是否需要重新处理的标记，以便后台任务获取到需要重新处理的文件。


# 文件解析之后的处理
文件解析之后，PDF和word文档中如果有图片，会返回图片列表，需要将图片保存到图片表中。




# 分片重新向量化的后台任务的工作流程
根据分片索引文本，生成文本向量，保存到向量表中。
